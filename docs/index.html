> 特性一览

[01]

任意视频格式

基于 FFMPEG 构建，支持几乎所有主流视频文件格式，无需预先转码。

[02]

随机数据破坏

使用 Mersenne Twister 算法在视频压缩数据中注入随机噪声，每次运行都会产生独一无二的视觉结果。

[03]

硬件加速解码

自动检测并启用 CUDA NVDEC 等硬件解码器，极大提升处理速度。

[04]

色彩空间转换

内置 YUV 到 RGB/BGR 的色域转换，通过 SWS 上下文保持色彩精度。

[05]

OpenCV 集成

可选的 OpenCV 支持，提供实时预览窗口和图像后处理能力，包括智能裁剪与缩放。

[06]

帧采样导出

可配置的帧采样间隔，自动将故障帧导出为 JPEG 图片序列，便于后期使用。

// 使用随机分布决定是否破坏当前帧的数据包
std::uniform_real_distribution<> odd(0, 1);
if (odd(gen) > 0.1) {
    // 随机选择破坏起点、长度和填充值
    std::uniform_int_distribution<> corruption_count(1, 6);
    std::uniform_int_distribution<> start(0, packet->size - 1);
    std::uniform_int_distribution<> val(0, 255);

    int random_start = start(gen);
    for (size_t i = 0, count = corruption_count(gen); i < count; i++) {
        for (int j = 0; j < random_length; j++) {
            packet->data[random_start + j] = val(gen);  // 注入噪声
        }
    }
}

> 三分钟上手

▌ 需要 FFMPEG 4.3.3+ / OpenCV 4.5.4+ / CMake 3.16+ / C++17
01

克隆仓库

将项目代码克隆到本地开发环境。

$ git clone https://github.com/yinguobing/make-it-glitch.git
02

CMake 构建

使用 CMake 配置并编译项目，生成可执行文件。

$ cd make-it-glitch && mkdir build && cd build $ cmake .. $ make
03

运行故障处理

指定输入视频和输出目录，开始生成故障艺术画面。

$ ./glitch your-video.mp4 output-images/
04

导出原始帧（可选）

添加额外参数可导出未经破坏的原始视频帧作为对比参考。

$ ./glitch your-video.mp4 output-images/ no-touching
